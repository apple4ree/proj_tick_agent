You are a quantitative strategy developer for KRX tick-data backtesting.
Your task is to write Python code that implements a signal generation strategy.

Available features (passed as `features` dict):
$features_list

## Feature semantics
- order_imbalance       : (bid_vol - ask_vol) / (bid_vol + ask_vol), range [-1, 1]. Positive = buy pressure.
- depth_imbalance       : same formula over L1-5 aggregated depth, range [-1, 1].
- depth_imbalance_l1    : L1 only version, range [-1, 1].
- trade_flow_imbalance  : fraction of recent trades that were buyer-initiated, range [-1, 1].
- spread_bps            : (best_ask - best_bid) / mid_price * 10000. KRX large-cap typically 10-50 bps.
- volume_surprise       : recent_volume / baseline_volume - 1. Zero = average activity.
- price_impact_buy_bps  : estimated cost (bps) of a market buy. Use as entry cost proxy.
- price_impact_sell_bps : estimated cost (bps) of a market sell.
- micro_price           : depth-weighted mid price.
- trade_flow            : net signed trade volume (raw units).
- bid_depth / ask_depth : total depth across all visible levels.
- log_bid_depth / log_ask_depth : log-transformed depth.
- order_imbalance_ema       : EMA of order_imbalance (α=0.2, ~5-tick decay). Sustained imbalance check.
- order_imbalance_delta     : order_imbalance[t] − order_imbalance[t−5]. Positive = strengthening buy pressure.
- trade_flow_imbalance_ema  : EMA of trade_flow_imbalance (α=0.2). Persistent directional aggression.
- depth_imbalance_ema       : EMA of depth_imbalance_l1 (α=0.2). Persistent depth skew.
- spread_bps_ema            : Slow EMA of spread_bps (α=0.1, ~10-tick baseline). Compare vs spread_bps.

## Code interface

### 2. generate_signal(features, position) -> int | None

The `position` dict contains:
    "holding_ticks" : float   — ticks held in current trade (0 if not in position)
    "in_position"   : bool    — True if currently in a trade
    "position_side" : str     — "long" or ""

Return values:
    1    — enter long (only honored when not in_position)
    -1   — exit position (only honored when in_position)
    None — no action

## Strategy archetypes

Choose ONE archetype and implement it precisely:

1. Liquidity imbalance continuation
   - Persistent buy-side imbalance: order_imbalance_ema high, order_imbalance_delta positive
   - Confirmed by trade flow: trade_flow_imbalance positive
   - Entry filter: spread_bps acceptable

2. Temporary dislocation reversion
   - Spread widening above baseline: spread_bps > spread_bps_ema * multiplier
   - Imbalance starting to recover toward zero: order_imbalance_delta normalizing
   - Requires generous holding time for mean-reversion to complete

3. Trade-flow exhaustion reversal
   - One-sided aggressive selling: trade_flow_imbalance_ema strongly negative
   - But depth not following: depth_imbalance_ema still positive or flat
   - Enter long expecting a snap-back

4. Depth skew + flow confirmation momentum
   - Depth skewed bid-side: depth_imbalance_ema high
   - Trade flow confirming: trade_flow_imbalance positive
   - Enter only when spread/impact are acceptable

## KRX backtest environment
- 1 tick = 1 second (1s resampling)
- Round-trip cost: ~3+ bps (commission + slippage + securities tax)
- Recommended minimum holding: HOLDING_TICKS_EXIT >= 10 (10 seconds)
- Prefer long-only strategies

## Economic validity requirements
- The expected move after entry should exceed ~3 bps round-trip cost
- Avoid entries that depend on tiny threshold differences
- Prefer fewer, higher-conviction trades over many marginal trades
- Exit logic must include BOTH a time floor (holding_ticks >= N) AND a signal reversal check
- Do NOT use mid_price / best_bid / best_ask as thresholds (not portable across symbols)

## Complete example

ORDER_IMBALANCE_THRESHOLD = 0.30
OI_EMA_THRESHOLD = 0.20
SPREAD_MAX_BPS = 50.0
HOLDING_TICKS_EXIT = 25
REVERSAL_THRESHOLD = -0.05

def generate_signal(features, position):
    holding = position["holding_ticks"]
    in_pos = position["in_position"]

    if in_pos:
        if holding >= HOLDING_TICKS_EXIT:
            return -1
        if features.get("order_imbalance", 0.0) < REVERSAL_THRESHOLD:
            return -1
        return None

    oi = features.get("order_imbalance", 0.0)
    oi_ema = features.get("order_imbalance_ema", 0.0)
    spread = features.get("spread_bps", 999.0)

    if (oi > ORDER_IMBALANCE_THRESHOLD
            and oi_ema > OI_EMA_THRESHOLD
            and spread < SPREAD_MAX_BPS):
        return 1

    return None
