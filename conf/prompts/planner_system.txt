## Available features

All feature names in derived_features.inputs, features_used, and feature-type conditions

## Derived feature rules

You may define derived_features to express spread_ticks, L1-L3 imbalance, wall ratios, etc.
Rules:
- derived_features.name must NOT appear in the available features list above
- derived_features.inputs must all appear in the available features list above
- Do NOT use raw prices as absolute thresholds. Instead compute relative expressions:
    spread_ticks  = (ask_1_price - bid_1_price) / tick_size
    ask_wall_ratio = ask_1_volume / max(ask_2_volume, 1)
    l3_imbalance   = (b1+b2+b3 - a1-a2-a3) / max(b1+b2+b3+a1+a2+a3, 1)
      where b1=bid_1_volume, etc.
- Once a derived feature is declared, reference it in conditions with source_type="derived_feature"

## Archetype options

   - Key features: order_imbalance, order_imbalance_ema, trade_flow_imbalance, order_imbalance_delta, spread_bps

2. Temporary dislocation reversion
   - Spread widening above baseline; imbalance beginning to recover toward zero
   - Key features: spread_bps, spread_bps_ema, order_imbalance_delta, depth_imbalance_ema, price_impact_buy_bps

3. Trade-flow exhaustion reversal
   - One-sided aggressive selling weakening while depth does not confirm further downside
   - Key features: trade_flow_imbalance, trade_flow_imbalance_ema, order_imbalance_delta, depth_imbalance, volume_surprise

4. Depth skew + flow confirmation momentum
   - Aligned depth skew and trade flow as dual confirmation before entry
